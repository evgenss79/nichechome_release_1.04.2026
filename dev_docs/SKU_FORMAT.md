# SKU Format Documentation

## SKU Key Format

All SKU keys in `data/stock.json` and `data/branch_stock.json` must follow the format generated by the `generateSKU()` function located in `includes/helpers.php`.

### SKU Format Pattern

```
{PREFIX}-{VOLUME}-{FRAGRANCE}
```

Where:
- **PREFIX**: 2-3 character product type identifier
- **VOLUME**: Volume code (numeric for measured volumes, "STA" for standard)
- **FRAGRANCE**: First 3 characters of fragrance name (uppercase)

### Product Prefixes

| Product Type | Prefix | Example |
|-------------|--------|---------|
| diffuser_classic | DF | DF-125-CHE |
| candle_classic | CD | CD-160-BEL |
| home_spray | HP | HP-50-SAN |
| car_clip | CP | CP-STA-AFR |
| textile_spray | TP | TP-STA-BAM |
| limited_new_york | LE | LE-270-NEW |
| limited_abu_dhabi | LE | LE-270-ABU |
| limited_palermo | LE | LE-270-PAL |
| aroma_sashe | ARO | ARO-STA-CHE |
| christ_toy | CHR | CHR-STA-EDE |

### Volume Codes

- **Numeric volumes**: Remove "ml" suffix → `125ml` becomes `125`
- **Standard volume**: Use `STA` → `standard` becomes `STA`

Examples:
- `125ml` → `125`
- `250ml` → `250`
- `500ml` → `500`
- `standard` → `STA`

⚠️ **IMPORTANT**: Never use `STD` for standard volumes. Always use `STA`.

### Fragrance Codes

Fragrance codes are the first 3 uppercase characters of the fragrance name:

| Fragrance | Code |
|-----------|------|
| africa | AFR |
| abu_dhabi | ABU |
| bamboo | BAM |
| bellini | BEL |
| blanc | BLA |
| cherry_blossom | CHE |
| dubai | DUB |
| eden | EDE |
| fleur | FLE |
| lime_basil | LIM |
| new_york | NEW |
| palermo | PAL |
| rosso | ROS |
| santal | SAN |
| tobacco_vanilla | TOB |

⚠️ **IMPORTANT**: Always use the first 3 characters of the fragrance identifier, not abbreviations:
- ✅ Correct: `new_york` → `NEW` (not `NYC`)
- ✅ Correct: `eden` → `EDE` (not `EDN`)

## Complete SKU Examples

### Limited Edition Products
```json
"LE-270-NEW": {
    "productId": "limited_new_york",
    "volume": "270ml",
    "fragrance": "new_york",
    "quantity": 10,
    "lowStockThreshold": 3
}
```

### Car Perfume (Standard Volume)
```json
"CP-STA-AFR": {
    "productId": "car_clip",
    "volume": "standard",
    "fragrance": "africa",
    "quantity": 30,
    "lowStockThreshold": 10
}
```

## Validation

To verify all SKUs in your stock files are correctly formatted, run:

```bash
php /tmp/test_sku_validation.php
```

This will check:
1. All SKUs match `generateSKU()` output
2. Required SKUs exist
3. No old/incorrect SKU formats are present
4. All branch_stock.json SKUs reference valid stock items

## Historical Changes

### 2024-12 - SKU Format Corrections

The following corrections were made to align with `generateSKU()` function:

1. **Limited Edition New York**: `LE-270-NYC` → `LE-270-NEW`
   - Reason: Use first 3 chars of `new_york` = `NEW`, not city abbreviation

2. **Car Clips**: `CP-STD-*` → `CP-STA-*`
   - Reason: Standard volume should be `STA`, not `STD`
   - Affected: `CP-STD-AFR` → `CP-STA-AFR`, `CP-STD-DUB` → `CP-STA-DUB`

3. **Textile Sprays**: `TP-STD-*` → `TP-STA-*`
   - Reason: Standard volume should be `STA`, not `STD`
   - Affected: `TP-STD-BAM` → `TP-STA-BAM`, `TP-STD-FLE` → `TP-STA-FLE`

4. **Diffuser Eden**: `DF-STA-EDN` → `DF-STA-EDE`
   - Reason: Use first 3 chars of `eden` = `EDE`, not custom abbreviation

5. **Home Spray Santal**: Added `HP-50-SAN`
   - Reason: Missing product variant that should be available

## Implementation

The SKU generation is implemented in two places:

1. **PHP Backend**: `includes/helpers.php::generateSKU()`
   - Used for server-side stock validation and checkout
   
2. **JavaScript Frontend**: `assets/js/app.js::generateSKU()`
   - Used for client-side cart management

Both implementations must remain synchronized to ensure consistency.
