# Исправление Email Админки - Инструкция для Пользователя

## ✅ Что было исправлено

### 1. Исправлена ошибка HTTP 500
- Админка `/admin/email.php` больше не падает с ошибкой 500
- Если произойдет фатальная ошибка, вы увидите понятное сообщение вместо белой страницы
- Все ошибки PHP логируются в `/logs/php_errors.log`

### 2. Создана папка логов
- Создана защищенная папка `/logs` для хранения логов
- Логи недоступны через браузер (защита .htaccess)
- Автоматическая ротация логов (при превышении 5MB)

### 3. Улучшенная диагностика SMTP
- Тест соединения теперь показывает пошаговую проверку:
  1. Проверка конфигурации
  2. Проверка DNS (преобразование домена в IP)
  3. Проверка доступности порта
  4. Расшифровка пароля
  5. Подключение и авторизация SMTP
- Понятные сообщения об ошибках на русском языке

### 4. Расширенная вкладка Логи
- Теперь показывает **два типа логов**:
  - Email логи (попытки отправки писем)
  - PHP логи (ошибки и предупреждения PHP)
- Показывает статус папки логов (существует/доступна для записи)

### 5. Предупреждение о несоответствии From Email
- Если "From Email" не совпадает со "SMTP Username", появляется предупреждение
- Важно: tuthost требует, чтобы From Email = SMTP Username

## 📋 Инструкция по проверке (2-3 шага)

### ШАГ 1: Откройте админку Email

1. Откройте браузер
2. Перейдите на: `https://nichehome.ch/admin/email.php`
3. **Проверка:** Страница открылась без ошибки 500? ✓

### ШАГ 2: Проверьте логи

1. Нажмите вкладку **Logs** (Логи)
2. **Проверка:** Видите зеленое сообщение "Logs directory is properly configured and writable"? ✓
3. **Проверка:** Видите две секции: Email Logs и PHP Error Logs? ✓

### ШАГ 3: Настройте SMTP (если еще не настроено)

1. Перейдите на вкладку **SMTP Settings**
2. Заполните настройки tuthost:
   ```
   SMTP Host: mail.nichehome.ch
   SMTP Port: 587
   Encryption: TLS
   SMTP Username: mailer@nichehome.ch (ваш email)
   SMTP Password: [ваш пароль]
   From Email: mailer@nichehome.ch (ДОЛЖЕН совпадать с Username!)
   From Name: NicheHome.ch
   ✓ Enable email sending (включить отправку)
   ```
3. Нажмите **Save SMTP Settings**
4. **Проверка:** Появилось сообщение "SMTP settings saved successfully"? ✓

### ШАГ 4: Протестируйте соединение

1. Перейдите на вкладку **Test Email**
2. Прокрутите вниз до **Test SMTP Connection**
3. Нажмите кнопку **Test Connection**
4. **Ожидаемый результат:**
   ```
   ✓ Configuration Check: Configuration is valid
   ✓ DNS Resolution: Resolved to IP: xxx.xxx.xxx.xxx
   ✓ Port Connectivity: Port 587 is reachable
   ✓ Password Decryption: Password decrypted successfully
   ✓ SMTP Connection & Auth: Connection and authentication successful
   
   ✓ Your SMTP configuration is working correctly!
   ```

### ШАГ 5: Отправьте тестовое письмо

1. На той же вкладке **Test Email**
2. Введите ваш email адрес
3. Нажмите **Send Test Email**
4. **Проверка:** Получили письмо на почту? ✓
5. Перейдите на вкладку **Logs**
6. **Проверка:** Видите новую запись `[SUCCESS] Event: test`? ✓

## 🔍 Что смотреть при проблемах

### Если не работает отправка писем:

1. **Откройте Admin → Email → Logs**
2. **Посмотрите на записи с `[FAILED]`**
3. **Прочитайте текст ошибки** - он скажет что не так

### Если Test Connection падает:

- **✗ DNS Resolution** → Проверьте написание hostname (mail.nichehome.ch)
- **✗ Port Connectivity** → Проверьте порт (587 для TLS, 465 для SSL) и firewall
- **✗ Password Decryption** → Пересохраните пароль в SMTP Settings
- **✗ SMTP Connection & Auth** → Неверный логин/пароль, или неправильный тип шифрования

### Если письма не приходят клиентам:

1. Проверьте что **Enable email sending** включено (галочка стоит)
2. Проверьте **Admin → Email → Logs** - есть ли ошибки?
3. Проверьте что **From Email = SMTP Username** (требование tuthost)
4. Проверьте папку СПАМ у клиента

## 🐛 Debug Mode (режим отладки)

Если нужны подробные логи SMTP:

1. Откройте **Admin → Email → SMTP Settings**
2. Найдите **Debug Mode** (желтый блок вверху)
3. Поставьте галочку **Enable SMTP Debug Mode**
4. Нажмите кнопку **Enable Debug**
5. Теперь при тестах будут показываться подробные логи SMTP

**⚠️ Важно:** После отладки **выключите Debug Mode** - он показывает много технических деталей.

## 📁 Где находятся логи

Логи хранятся на сервере в папке `/logs` (рядом с `/admin` и `/includes`):

- `/logs/email.log` - Все попытки отправки email (успешные и неудачные)
- `/logs/php_errors.log` - PHP ошибки и предупреждения
- `/logs/email_YYYY-MM-DD_HH-mm-ss.log` - Архивные логи (автоматически создаются при превышении 5MB)

**Доступ к логам:**
- Через админку: **Admin → Email → Logs**
- Через FTP/SSH: скачайте файлы из папки `/logs`
- ⛔ **Логи недоступны через браузер** (защищены .htaccess)

## 🔒 Безопасность

✅ Пароли шифруются (AES-256-CBC)
✅ Пароли в логах заменены на `***`
✅ Пароли не показываются в админке
✅ Папка логов защищена от публичного доступа
✅ CSRF защита на всех формах

## ⚠️ Важные правила

1. **From Email ОБЯЗАН совпадать с SMTP Username** (требование tuthost)
2. **Никому не сообщайте SMTP пароль** (он зашифрован в базе)
3. **Регулярно проверяйте логи** чтобы видеть проблемы с отправкой
4. **Debug Mode включайте только при отладке** (потом выключайте)

## 📞 Что делать если не получается

### Вариант 1: Смотрим логи

1. Откройте **Admin → Email → Logs**
2. Найдите красные записи `[FAILED]`
3. Прочитайте текст ошибки
4. Исправьте проблему согласно ошибке

### Вариант 2: Включаем Debug Mode

1. **Admin → Email → SMTP Settings**
2. Включите **Debug Mode**
3. Попробуйте отправить тест
4. Смотрите подробные логи SMTP
5. Выключите Debug Mode после

### Вариант 3: Проверяем с tuthost

Если ошибка содержит:
- `Authentication failed` → Неверный логин/пароль
- `Connection refused` → Порт заблокирован или неверный
- `TLS handshake failed` → Неверный тип шифрования (должен быть TLS для порта 587)

Свяжитесь с поддержкой tuthost:
- Убедитесь что SMTP включен для вашего аккаунта
- Проверьте квоты на отправку писем
- Попросите проверить IP репутацию сервера
- Спросите про настройку SPF/DKIM

## ✅ Root Cause (Корень проблемы)

**Главная причина HTTP 500:**
- Отсутствовала папка `/logs` → попытка записи в несуществующую папку давала fatal error

**Почему письма не уходили:**
- Проверьте настройки SMTP (логин/пароль/хост/порт)
- Проверьте что From Email = SMTP Username
- Проверьте логи на наличие конкретных ошибок

**Что было сделано:**
1. Создана защищенная папка `/logs` с .htaccess
2. Добавлен обработчик фатальных ошибок (register_shutdown_function)
3. Все require_once теперь проверяют существование файла
4. Улучшена диагностика SMTP (5 шагов проверки)
5. Логи доступны через админку
6. Добавлены предупреждения о проблемах конфигурации
